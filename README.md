Explication du code :

HTML :
Le code html est écrit en html 5 qui est la version la plus récente du html. On indique que le site est en français grâce à un attribut lang dans la baslise html. Dans le header on peut trouver un meta contenant le type de caractère que l'on peut retrouver dans le code ou dans les textes : il s'agit de l'utf-8. Ensuite on voit des balises meta qui servent à : être compatible avec le navigateur, être mobile-first, introduire la favicon (symbole du site), intégrer un code css qui permet d'enlever tout les styles par défaut qui sont appliqués avant, intégrer également le code css spécifique au site, accéder à la librairie jquery avec un link, et utiliser cette dite librairie. Dans le body, j'y ai mis principalement 3 balises : le header, le main, et le footer. Le header contient le titre. Dans le main on peut y trouver un nav servant à naviguer dans des liens internes du site dits ancres. La section intro ayant son importance dans le code css pour la mise en forme, je l'ai donc intégrée dans une div avec la nav. Toutes les autres sections sont constituées de la même manière : c'est-à-dire d'un h2 ainsi qu'un paragraphe composé de 5 sections de lipsum (lorem ipsum : texte pour combler le manque de contenu dans les sites). Chaque section a un identifiant, permetant de saisir directement la section concernée dans l'intro. Le footer est composé de 3 containers grosso modo. Le premier contient des links vers les réseaaux sociaux, le second contient des élèments permettant de me contacter personnelement par mail, c'est un mailto. Le dernier contient les cgu que j'ai téléchargé sur un site internet spécialisé en législation après avoir complété les informations demandées. Le body et le code html dans son ensemble se termine là.

CSS :
Dans le css, je détaille déjà le h1, avec la position, la taille de la police, la couleur du background, je l'ai mis en display flex afin de pouvoir utiliser les propriétés affiliées permettant le positionnement tels que justify-content et align-items, je met aussi un taille globale du block et impose le fait que les lettres du titres vont à la ligne si elles dépassent. Dans le body j'ai décidé d'assombrir le thème en mettant un background gris. La taille des lettres est de 20 px, et je choisis d'aligner tous les texte en justify. Concernant la nav j'ai mis des padding ainsi qu'une bordure en bas et des margin. Dans le footer j'ai décidé de reprendre le thème principal d'un background noir et d'un color blueviolet ainsi que des margin pour le reste du footer. Quant aux images, je leur ai mis a toutes une taille petite pour les manipuler plus facilement et le radius qui accompagne leur border. Ensuite j'utilise des breakpoints qui sont à 600px, 768px, 992px. Les breakpoints que j'ai créée grâce aux media queries sont assez standards, ils break à min : 600, 768; 992. Je n'en ai pas ajouté plus car il n'est nul besoin de viser un format de screen qui serait supérieur. Lors dès média queries on indique le only pour la simple et bonne raison que les anciens navigateurs web ne prennent pas en compte le only et donc la suite du code des médias qui pourrait créer des bugs si le navigateur est trop vieux. Le code qui s'applique ensuite n'est compté que pour les "nouveaux navigateurs". On indique donc que la taille minimum du media est égal à 600px de largeur dans un premier temps. J'ai donc décidé de monter les px de la height du h1 et du h2. J'ai également changé les paramètres du anv ainsi que le padding or margin des p contenus dans main and footer. A parir de 768px j'ai augmenté consi$dérablement le font-size du h1 et h2 ainsi que certains des autres tels que les classes : contact, cgu, main p, ul, nav. Ayant également créé de ouvelles classes tels que flex_footer, .nav .nav_intro. Dans les media min width = 992px l'on peut voir très peut de modifications...
